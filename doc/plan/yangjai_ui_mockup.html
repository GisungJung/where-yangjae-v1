의사결정 페이지
  • 지도 200px로 상단 hero — 카카오맵 SDK 로딩 전 회색 그리드 fallback.
  • 평균 평점 + 분포 미니 그래프로 평가의 신뢰도 시각화 (단순 평균만으로는 판단 부족).

  • Floating CTA "평가하기"는 backdrop-blur 처리 — 스크롤해도 항상 노출.

08폐업 확인 다이얼로그 /restaurant/:id

2단계 확인 흐름 — 액션시트 → 모달
  • 2단계 UX — 액션 시트(02 참고)에서 "폐업으로 표시" 탭 → 이 다이얼로그가 뜸. 되돌리기 부담스러운 상태 전환이라 명시적 확인 필수.
  • iOS native alert 패턴 — 중앙 정렬 280px 카드, backdrop-filter: blur(16px), 어두운 scrim(0.5)으로 배경 디그레이드.
  • 버튼 배치 컨벤션 — 안전한 "취소" 좌측 (primary 약체), 위험한 "폐업 처리" 우측 (danger red, bold) — iOS HIG 그대로.
  • 메시지에 영향 범위 명시 — "12건 평가는 보존됩니다"로 데이터 손실 없음을 안심시킴, 실수 부담 ↓.
  • 휴업 처리는 다이얼로그 생략 — 가벼운 액션(되돌리기 쉬움)이라 토스트만으로 피드백. 폐업만 이 흐름.
  • "운영중으로 복구"도 다이얼로그 없음 — 액션 시트에서 즉시 적용 + 토스트.

09빈 상태 — 검색 결과 없음 /?q=:keyword

Empty state — 막다른 길에서 다음 행동으로 잇기
  • 일러스트 + 명확한 사유 + 다음 행동 3단 구조. 회색 돋보기 + 주황 X 마크로 "검색했지만 못 찾음"을 비언어적으로 전달.
  • 1차 액션 "양꼬치 등록하기" — 빈 결과를 컨텐츠 추가 동기로 전환 (구글시트 컨셉과 일치).
  • 2차 액션 "검색어 지우기" — 빠른 복귀 경로.
  • 하단 카테고리 칩 제안 — 검색 의도를 카테고리 탐색으로 우회.
  • 변형 케이스 (이 패턴 재사용):
    • 첫 방문 (식당 0개) — "아직 등록된 식당이 없어요" + 큰 ＋ 버튼만, 칩 섹션 생략
    • 평가 0개 (상세) — 한줄평 섹션 자리에 "첫 평가를 남겨보세요" + 별 픽커 inline
    • 룰렛 풀 0개 — "조건에 맞는 식당이 없어요. 필터를 줄여주세요" + "조건 변경" CTA
    • 네트워크 오류 — "연결 실패" + "다시 시도" outline 버튼

10스켈레톤 로더 (초기 로딩)

데이터 fetching 0.3–1.0s 구간 UX
  • 실제 카드 레이아웃 미러링 — 카드 형태·뱃지 개수·텍스트 줄 수까지 동일하게 placeholder. Content-aware skeleton.
  • 왼→오 shimmer 1.4s 루프 — opacity 펄스보다 활동성 높고 피로감 적음. 색상은 #ECEEF1 베이스에 white 50% 그라데이션.
  • 헤더·탭바·검색 입력 같은 네비 영역은 즉시 실 콘텐츠 — 인지 부담 줄이고 앱 정체성 유지.
  • 3–4장 정도만 표시. 무한 placeholder는 오히려 답답함 유발.
  • 스켈레톤 → 실 카드 전환은 200ms fade-in 권장 (정적 목업에선 표현 생략).
  • 로딩이 1초 초과 시 "잠시만요 ☕" 토스트 보조 노출 추천.
  • 적용 화면: 홈 진입, 상세 진입(지도 + 카드 영역), 룰렛 후보 풀 계산 등.

11Pull-to-refresh /

상단에서 끌어 새로고침 — 활성 상태
  • 상단 56px 인디케이터 노출, 회전 스피너(0.9s loop) + Primary 컬러 텍스트.
  • 3단계 상태 —
    • (1) 드래그 중 (60px 미만): "당겨서 새로고침" + 정지된 화살표
    • (2) 임계점 초과 (60px+): "놓아서 새로고침" + 화살표 180° 회전
    • (3) 새로고침 중 (이 목업): 스피너 회전 + "새로 불러오는 중…"
  • 하단 카드 opacity 0.75 → 0.35 그라데이션 — 새 데이터 도착 전임을 시각적으로 암시. 실 구현에선 그대로 두는 게 일반적이지만 목업 전달용으로 표현.
  • 임계점 보통 60–80px 드래그 — 너무 짧으면 오발, 너무 길면 피로.
  • iOS·Android 네이티브 PTR과 동등한 햅틱 피드백 권장 (web vibration API 활용).
  • 실패 시 "새로고침 실패. 네트워크 확인" 토스트 + 인디케이터 사라짐 + 기존 데이터 유지.
  • 적용 화면: 홈 리스트, 상세 평가 섹션, 룰렛 결과 (다시 굴리기와 별개로).
