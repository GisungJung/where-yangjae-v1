## Task #1 — 홈 리스트 10개 단위 페이징

> 기획서 §8.1. 현재 구현(`HomePage.tsx`)은 전체 목록 일괄 렌더. dev가 페이지 단위 렌더로 전환 후 검증.

### 1-A 정상 페이징 — 첫 페이지

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | `/` 접속 (필터·검색 초기 상태) | 카드 목록이 최대 **10개**만 렌더됨 | ☐ |
| 2 | 페이지 하단 페이지네이션 영역 확인 | "1 / N" 또는 이전/다음 버튼 표시 | ☐ |
| 3 | "총 N건" 카운트 확인 | 전체 식당 수(필터 결과 수)와 일치 | ☐ |
| 4 | "다음" 버튼 탭 | 11번째~ 식당 카드로 갱신, 스크롤 최상단 이동 | ☐ |
| 5 | "이전" 버튼 탭 | 1번째~ 카드로 복귀 | ☐ |
| 6 | 첫 페이지에서 "이전" 버튼 상태 확인 | 비활성(disabled) 또는 숨김 | ☐ |

### 1-B 끝 페이지

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 마지막 페이지로 이동 (페이지 번호 or 반복 "다음") | 마지막 페이지 카드 1~10건 렌더 | ☐ |
| 2 | "다음" 버튼 상태 확인 | 비활성(disabled) 또는 숨김 | ☐ |
| 3 | 식당 수가 10의 배수일 때 마지막 페이지 | 카드 정확히 10개 렌더 | ☐ |
| 4 | 식당 수가 10의 배수+N일 때 마지막 페이지 | 카드 N개만 렌더 | ☐ |

### 1-C 검색 결과 페이징

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 검색창에 "국" 입력 | 10개 이하 결과: 페이지네이션 미표시 또는 1페이지만 | ☐ |
| 2 | 검색창에 "식" 입력 (결과 >10 예상) | 첫 10개만 렌더, 페이지네이션 표시 | ☐ |
| 3 | 다음 페이지 이동 | 11번째~ 검색 결과로 갱신 | ☐ |
| 4 | 검색어 지우기(X) 클릭 | **1페이지로 초기화**, 전체 목록 첫 10개 렌더 | ☐ |

| 6 | 카테고리 chip 변경 | **1페이지로 초기화** | ☐ |
| 7 | 정렬 변경 ("평가 많은 순") | **1페이지로 초기화** | ☐ |

|------|------|-----------|------|

## Task #2 — 홈 화면 카카오맵

> 기획서 §5.2 "지도 뷰 (v2) — 전체 식당 핀 표시, 평점 색상 구분" + §8.1 맛집 목록. dev가 홈 상단/탭에 KakaoMapView 추가 후 검증.

### 2-A 마커 정상 표시

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | `/` 접속, 지도 영역 확인 | 카카오맵이 렌더됨 (회색 → 지도 타일 표시) | ☐ |
| 2 | 지도 로드 완료 후 마커 확인 | 좌표(`lat`, `lng`)가 있는 식당마다 마커 표시 | ☐ |
| 3 | 지도 확대/축소 | 마커 유지, 지도 정상 반응 | ☐ |
| 4 | 드래그 이동 | 마커 유지, 지도 이동 | ☐ |
| 5 | DevTools Network → 카카오맵 SDK 요청 확인 | `dapi.kakao.com` 요청 1회만 발생 (중복 로드 없음) | ☐ |

### 2-B 좌표 없는 식당

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | `lat = null`인 식당 확인 (시드 데이터) | 지도 마커 **미표시** | ☐ |
| 2 | 지도에 빈 공간이 생기는 경우 | 지도 중심은 양재역 기준 유지 | ☐ |
| 3 | 해당 식당의 목록 카드 확인 | 리스트 카드는 정상 표시 (마커 없어도 카드는 있음) | ☐ |

### 2-C 마커 클릭 → 상세 이동

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 지도 마커 탭 | 인포윈도우 또는 바텀시트에 식당명 표시 | ☐ |
| 2 | 식당명 / 카드 탭 | `/restaurants/:id` 상세 페이지로 이동 | ☐ |
| 3 | 뒤로가기 후 | 홈 지도 상태 유지 (재로드 없음이 이상적) | ☐ |
| 4 | 마커 여러 개 겹칠 때 | 탭 시 어느 한 식당으로 이동 (크래시 없음) | ☐ |

### 2-D 필터 연동

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 카테고리 chip "한식" 선택 | 지도 마커도 **한식 식당만** 표시 (또는 전체 유지 — dev 결정 기준으로 일관성 확인) | ☐ |
| 2 | "휴업·폐업 포함" 체크 OFF | 휴업·폐업 식당 마커 **미표시** | ☐ |
| 3 | "휴업·폐업 포함" 체크 ON | 휴업·폐업 마커도 **표시** (색상 또는 스타일 구분 권장) | ☐ |

### 2-E SDK 로드 실패 폴백 (기획서 §11.2)

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | DevTools → Network → `dapi.kakao.com` 차단 후 새로고침 | 지도 영역에 회색 폴백 박스 + 텍스트 안내 | ☐ |
| 2 | 폴백 상태 | 식당 리스트는 정상 렌더 (지도 실패가 앱 전체를 깨지 않음) | ☐ |
| 3 | 크래시 여부 | Error Boundary 발동 없음 | ☐ |

### 2-F 성능

| 단계 | 행동 | 기대 결과 | 판정 |

| 1 | Slow 3G에서 홈 진입 | 지도 로드 중 Skeleton(회색 박스) 표시 | ☐ |

## Task #3 — 식당 정보 수정 페이지

> 기획서 §8.4 등록 폼 기준 유추. dev가 `RestaurantDetailPage`의 ActionSheet "정보 수정" stub(`alert('곧 제공')`)을 실제 수정 페이지 또는 바텀시트로 구현한 후 검증.

### 3-A 수정 페이지 진입

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 상세 페이지 ⋯(더보기) 탭 | ActionSheet 열림 | ☐ |
| 2 | "정보 수정" 선택 | 수정 폼 페이지 또는 바텀시트로 이동/표시 | ☐ |
| 3 | 수정 폼 초기값 확인 | 현재 식당 데이터(이름·카테고리·시트·메뉴·비고·네이버URL)가 미리 채워짐 | ☐ |

### 3-B 필수 항목 검증

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 식당 이름(`name`) 비워두고 저장 | "식당 이름을 입력해 주세요" (또는 동등한 한국어 오류) inline 표시, 저장 차단 | ☐ |
| 2 | 카테고리 미선택 상태로 저장 | 오류 메시지 + 저장 차단 | ☐ |
| 3 | sheet_type(점심/저녁) 미선택 저장 | 오류 메시지 + 저장 차단 | ☐ |
| 4 | 필수 항목 모두 채운 후 저장 | 오류 없이 진행 | ☐ |

### 3-C 좌표 유효성

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 위도(`lat`) 입력란에 `999` 입력 저장 | "위도는 -90~90 범위여야 합니다" 오류 + 차단 | ☐ |
| 2 | 경도(`lng`)에 `200` 입력 저장 | "경도는 -180~180 범위여야 합니다" 오류 + 차단 | ☐ |
| 3 | 위도만 입력하고 경도 비워두기 | 오류 메시지 + 차단 (lat/lng은 쌍으로 입력) | ☐ |
| 4 | 위도·경도 모두 빈 값 (좌표 없음) | 허용 (좌표 NULL은 기획서 §7.3에서 허용) | ☐ |
| 5 | 양재역 좌표 `37.4841, 127.0343` 입력 | 유효성 통과, 저장 진행 | ☐ |
| 6 | 카카오 장소 검색으로 좌표 자동 채움 | 수동 입력 없이 자동 채워진 좌표로 저장 가능 | ☐ |

| 5 | 카테고리 변경 후 저장 | 상세 페이지 카테고리 배지 즉시 갱신 | ☐ |
| 6 | 좌표 변경 후 저장 | 상세 페이지 카카오맵 마커 새 좌표로 이동 | ☐ |

## Task #5 — '네이버지도에서 보기' 잔존 UI 확인

> 기획서 §5 "네이버 지도 API 미채택". dev가 관련 UI 제거 후, 잔존 요소가 없는지 검증.

### 현재 구현 기준 제거 대상 목록

아래 위치에서 네이버 지도 관련 UI를 검토한다.

| 위치 | 현재 요소 | 제거 후 기대 |
|------|-----------|-------------|
| `RestaurantDetailPage` ActionSheet | "네이버 지도에서 보기" 항목 (`naver_url` 있을 때) | **제거 또는 항목 미표시** |
| 상세 페이지 — 좌표 없을 때 폴백 섹션 | `<a>네이버 지도에서 보기 ↗</a>` 링크 | **제거** |
| 상세 페이지 — 좌표 있을 때 맵 아래 | `<a>네이버 지도에서 보기 ↗</a>` 링크 | **제거** |
| `restaurants` 테이블 `naver_url` 컬럼 | 데이터 보존 가능 (DB 컬럼 유지, UI만 제거) | 컬럼 삭제 불필요 |

### 4-C rate-limit — 1시간 윈도우 차단 (R3)

> `SET LOCAL`로 `now()`를 바꿀 수 없음(PostgreSQL 제약). 기존 행의 `created_at`을 service_role로 인위 조정 후 11번째 INSERT로 검증. (dba 확인)  
> **핵심**: 10건이 **1분 윈도우 밖·1시간 윈도우 안**에 분포해야 `rate_limit_hour`가 발동. 1분 안에 1건이라도 있으면 `rate_limit_minute`가 먼저 발동해 1시간 검증 불가.

**셋업 SQL (Studio SQL Editor — service_role):**

```sql
-- 테스트 reviewer의 기존 ratings 10건을 1분~1시간 사이로 분산
WITH target AS (
  SELECT id FROM public.ratings
   WHERE reviewer_id = '<test-reviewer-uuid>'
   ORDER BY id
   LIMIT 10
)
UPDATE public.ratings r
   SET created_at = now()
                  - interval '90 second'               -- 1분 윈도우 밖
                  - (random() * interval '50 minute')  -- 1시간 윈도우 안
 FROM target t
WHERE r.id = t.id;
```

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 위 SQL로 10행 `created_at` 조정 (1분~1시간 사이 분포) | UPDATE 10행 성공 | ☐ |
| 2 | UI에서 동일 reviewer로 11번째 평가 제출 시도 | **차단** — ERRCODE `54000`, hint `rate_limit_hour` | ☐ |
| 3 | UI 에러 메시지 | 1분 에러와 **구분 가능**한 안내 (예: "오늘 평가 한도에 도달했어요") | ☐ |
| 4 | `ratings` 행 수 | 10개 유지 (11번째 삽입 없음) | ☐ |
| 5 | 검증 후 원복 | `DELETE FROM ratings WHERE id IN (SELECT id FROM target)` 또는 `created_at` 복원 | ☐ |

| 3 | 상세 페이지 평균 평점 | **최신 평가 점수** 기준 avg 재계산 즉시 반영 (restaurant_stats 뷰 정규화) | ☐ |
| 4 | 한줄평 목록 | 같은 닉네임 평가 **2건 모두 표시** (최신순) | ☐ |

#### 4-H-4 rate-limit 메시지 — 트리거 원문 그대로

| 단계 | 행동 | 기대 결과 | 판정 |
|------|------|-----------|------|
| 1 | 1분 내 재평가 시도 (`rate_limit_minute`) | **"잠시 후 다시 시도해 주세요. 같은 닉네임은 1분에 1회만 평가할 수 있어요."** | ☐ |
| 2 | 1시간 10회 초과 (`rate_limit_hour`) | **"같은 닉네임은 1시간에 10회까지만 평가할 수 있어요. 잠시 후 다시 시도해 주세요."** | ☐ |
| 3 | 1분 경과 후 재시도 | 성공, 새 평가 삽입 | ☐ |
| 4 | 에러 후 폼 상태 | 입력값(별점·한줄평) 유지 | ☐ |
